# ygwang.info v4.1 — Codebase Summary

> Reference doc for the v5 redesign/reimplementation.

---

## Overview

A Hugo-based static personal site at **https://ygwang.info/**. Content is Simplified Chinese literary and creative work (诗歌、小说、杂谈、创意). Published by 王咏刚 (pen name 半轻人). Site title: **半轻小屋**.

---

## Stack & Build

| Item | Detail |
|------|--------|
| Generator | [Hugo](https://gohugo.io/) (no npm, no Makefile) |
| Theme | [hugo-PaperMod](https://github.com/adityatelange/hugo-PaperMod) (vendored under `themes/`) |

## Content Structure & Frontmatter

### Poems (`content/poems/`)

Flat files: `poem_0000.md` … `poem_0161.md` (162 total).

```toml
+++
author = '王咏刚'
title = '舞进尘埃'
date = 1995-05-01T00:00:00
draft = false
+++

<div class="poem">

```
<poem text here>
```

</div>
```

The poem body is always a fenced code block wrapped in `<div class="poem">`. The JS renderer detects this and applies character-level styling for traditional-form poems.

Section intro (`_index.md`) contains a banner image and an editorial note.

### Essays (`content/essays/`)

Each essay is a **subdirectory** containing a single `.md` file and any images:
```
essays/paris/paris.md
essays/paris/paris.jpg
```

No `_index.md` inside essay subdirectories — the `.md` file itself is the content page.

### Fictions (`content/fictions/`)

Same pattern as essays: subdirectory + `.md` file + images.

### Creations (`content/creations/`)

Same pattern as essays. Content often embeds external links (Suno AI, Mootion, Midjourney, etc.) with inline HTML allowed via `unsafe: true`.


### Poem Rendering (`poem_renderer.js` + `poem.css`)

The key custom feature. Every poem page embeds raw text inside a fenced code block inside `<div class="poem">`. On page load:

1. `PoemRenderer.render()` finds all `.poem` elements.
2. `isTraditionalPoem()` checks if ≥80% of punctuation-delimited sentences share a character length in {4, 5, 6, 7} — if so, it's a "traditional poem".
3. `renderTraditionalPoem()` replaces the `<code>` innerHTML: each Chinese character gets `<span class="letter">`, each punctuation gets a plain `<span>`, newlines become `<br>`.
4. CSS gives `.letter` a `border-bottom: 1px solid var(--underline-red)` (red underline per character).
5. Non-traditional poems render as plain preformatted text.

## What v5 Should Preserve

- **Content format**: TOML frontmatter + markdown + HTML is fine; all 160+ poems and other content must migrate cleanly.
- **URL structure**: existing URLs must not break (poem_0000 … poem_0161, essay/fiction/creation slugs).
- **Poem renderer logic**: the `isTraditionalPoem` heuristic and character-level underline rendering are a distinctive feature.
- **CJK typography**: `hasCJKLanguage`, east-Asian line break handling, `letter-spacing`.
- **Light/dark auto theme**: already working well.
- **No comments, no heavy JS**: site philosophy is minimal and fast.
